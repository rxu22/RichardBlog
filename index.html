Week 2 CS

Similar to last week, went over week 2 of CS 61A which consisted of chapters 1.5-1.6, and worked on Lab 1.

For the first lecture, control, I learned about the environment, frame, and the function’s relation to the environment.
The lecture did not go very in depth into the topic of environments since it will be covered in the next few lectures.
Environment - sequence of frames
Discussed how

  • an environment is either a global frame or a first (local) frame chained to a parent environment
  • every expression and statement is evaluated (executed) in an environment
  • expressions and subexpressions (pieces of an expression) are evaluated in the same environment as the statement or expression containing them

The lecture continues by defining some familiar programming terms:

  1. Def statements - bind a name to a function value in the first frame of the environment in which the def statement is executed
  2. Calling - a user-defined function creates a new local environment frame that binds the function’s formal parameters to the operand values (actual parameters) in the call
  3. Assigning - a variable binds a value to it in (for now) the first frame of the environment in which the assignment is executed

The lecture gave a quick overview on the topic of nested functions discussed in week 1, and connected it to the topic of environments
The Last topic of the lecture is over control with several new terms:

  1. Control expression - evaluates some or all of its operands in an order depending on the kind of expression, and typically on the values of those operands
  2. Statement - a construct that produces no value, but is used solely for its side effects
  3. Control statement - a statement that, like a control expression, evaluates some or all of its operands, etc.

Gave an example of conditional evaluation (if and else) - the most common kind of control
Suite - The sequence of indented statements after the colon
Went on to describe recursive function or loops
Introduced the while statement

The second lecture of the week covered higher-order functions.
The slides introduced some new terminology:

  1. domain - The set of possible argument values of a function
  2. range - The set of values that the function can return
  3. codomain - n is a set of values that includes the range, and possibly other values

They also provided an analogy that helped me understand the topic more: the square function has the real numbers as its domain, and the non-negative numbers as its range. We can choose to describe its codomain as the real numbers or as just the non-negative real numbers.

The following list of following terms is for documenting functions:

  1. precondition - should make clear what inputs are valid or under what conditions the function may be called
  2. postcondition -should make clear what the resulting output or effect of the function will be for correct inputs

These are all suggestions to use to provides enough information so that a programmer can use the function properly and understand what it does without having to read its body

The higher order function that they showed for this slide was the lamda function which is an abbreviation. The function also has no return.
Gave several examples:

  • summation(10, lambda x: x**3) which returns the sum of cubes
  • summation(10, lambda x: 1 / x) harmonic series
  • summation(10, lambda k: x**(k-1) / factorial(k-1)) approximate e**x

Gave another term: First-class values- can assign them to variables, pass them to functions, and return them from functions

The last lecture of the week goes in depth on the topic of environments with several exercises, which was discussed in the first lecture of week 2.
